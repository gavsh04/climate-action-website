Home page

Technical Description

The Home Page utilizes semantic HTML5 tags like "header", "nav"", and "main" to provide a clear and logical document structure. The layout is styled using home.css and a global style.css, where Flexbox is employed to create a responsive navigation bar and align climate topic cards. A high-impact "Hero" section uses a CSS linear gradient overlay on a background image to ensure text readability. Interactivity is added through a sticky header and scroll-reveal animations controlled by script.js, which toggles CSS classes as elements enter the viewport.

Accessibility

Readability is prioritized through high color contrast, specifically using white text on a dark green (#2E7D32) background. Semantic landmarks and descriptive alt text for images (e.g., "Solar panels on a roof") ensure that users with screen readers can navigate the site effectively.

Gallery Page

Technical Description

The gallery grid is implemented using CSS Grid with the repeat(auto-fit, minmax(250px, 1fr)) property, allowing the layout to automatically adjust based on the screen width. A custom Lightbox modal was developed for the "Extended View" requirement. This modal uses absolute positioning and a high z-index to overlay the content. JavaScript in gallery.js handles the logic for opening the modal, dynamically populating it with data from HTML data-* attributes (like data-description), and toggling color schemes or font styles.

Accessibility

The modal includes aria-hidden attributes to manage visibility for assistive technologies. Keyboard accessibility is supported by allowing users to close the modal using the "Escape" key, and every gallery item is interactive with clear focus states.

Challenges and Lessons Learned


Managing global vs. page-specific styles during the team merge. Initially, teammate styles were overwriting the shared navigation colors.

I established a strict "Global CSS" (style.css) for shared components like the footer and nav, while requiring teammates to use unique class prefixes in their own CSS files.

I learned the importance of modularity in JavaScript. Moving the lightbox logic from the main script.js to a dedicated gallery.js made the code easier to maintain and faster to load on non-gallery pages.
