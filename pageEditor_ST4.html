Page 1

User Profile Page

Technical Description

The User Profile page (profile.html) is driven by JavaScript prompt() calls that collect user data across three stages. Data is stored in a profileData object and rendered via DOM manipulation. I implemented conditional logic to handle skipped prompts, ensuring that incomplete data does not display "null" to the user.

Accessibility

The progress bar uses ARIA role="progressbar" with aria-valuenow attributes. Dynamic content updates are announced via aria-live regions to support screen reader users.

Page 2

Sitemap Page

Technical Description

The sitemap (sitemap.html) uses an inline SVG element for full responsiveness. Each node is a clickable <rect> wrapped in an SVG anchor tag. I used CSS transitions to provide visual feedback when users hover over site nodes.

Accessibility

The SVG element carries a role="img" attribute and an aria-label describing the diagram's purpose. Individual nodes use SVG <title> tags for screen-reader labels.

Page 3

Content Page — Ocean Health

Technical Description

This page (content_ST4.html) utilizes CSS Grid for a clean, five-section layout. I implemented a green-tinted hero banner overlay to match the team's visual identity. The page includes a data table regarding ocean acidification milestones, styled for maximum readability.

Challenges and Lessons Learned

SVG Scaling: Making the sitemap responsive required using viewBox rather than fixed widths. I learned that SVG elements require specific CSS targeting (like fill instead of background-color).

Anchor Offsets: A major challenge was the sticky header covering titles. I solved this using scroll-margin-top, a technique I shared with the group to ensure consistency.

Compliance

All pages implemented for the 'Clime' project comply with JANET regulations. The content is educational, factual, and directly relevant to SDG 13 (Climate Action). All images are royalty-free (Unsplash/iStockPhoto) and properly referenced. No personal data is stored server-side; the User Profile exists only within the local browser session.
