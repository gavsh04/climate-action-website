User Profile Page

Technical Description

The User Profile page (profile.html) was built using the shared HTML template structure, with a <header>, <main>, and <footer>. The core interaction is driven entirely by JavaScript prompt() calls that collect user data step-by-step across three distinct stages: Basic Details, Interests & Preferences, and Climate Actions & Goals.

Data is stored in a JavaScript object (profileData) that persists across steps during a session. DOM manipulation via getElementById() and innerHTML is used to progressively reveal profile sections and update a progress bar as each step is completed. Conditional logic handles skipped prompts — skipped fields are omitted from the displayed output, and the progress indicator reflects partial completion. Users can revisit and complete skipped steps using dedicated "Revisit Step" buttons, which re-run only the missing prompts and merge the new data with what was already collected.

Embedded CSS within a <style> block styles the step buttons, progress bar, and profile output area. These styles are page-specific and do not conflict with the Global CSS. The page also uses the active class on the navigation link for Profile to meet the active-page requirement.

Accessibility

All dynamically generated profile content uses clear, descriptive labels so that screen readers can interpret the output. The progress bar uses an ARIA role="progressbar" attribute with aria-valuenow, aria-valuemin, and aria-valuemax to communicate completion status. Buttons use meaningful text labels (e.g. "Start Profile", "Revisit Step 1") rather than generic icons. Font size and contrast ratios were kept above WCAG AA minimums.

Sitemap Page

Technical Description

The Sitemap page (sitemap.html) uses an inline SVG element embedded directly within the <main> section. The SVG uses a viewBox attribute and width="100%" / height="auto" to make it fully responsive across different screen sizes.

Each page in the website hierarchy is represented as a rounded rectangle (<rect>) with a <text> label. Nodes are connected using <line> elements to communicate the parent–child structure visually. Every node is wrapped in an SVG <a> tag linking to the corresponding page file, making each node clickable. Hover effects are implemented via CSS targeting SVG elements (e.g. a:hover rect), changing fill colour to provide visual feedback.

Each node group contains an SVG <title> element that provides an accessible name for screen readers. Focusable nodes support keyboard Tab navigation and show a visible focus outline via CSS.

Accessibility

The SVG element carries a role="img" attribute and an aria-label describing the diagram's purpose. Individual nodes use SVG <title> tags for screen-reader labels. All links are keyboard-navigable via Tab, and a visible focus state (stroke highlight) is applied. Text contrast within nodes meets WCAG AA guidelines.

Content Page — Ocean Health & Climate Action

Technical Description

The Content Page (content_ST4.html) covers the topic of Ocean Health and its relationship to climate change. It follows the shared HTML template and links to both the Global CSS (style.css) and a dedicated page-specific stylesheet (style_ST4.css).

The page is divided into five clearly-labelled sections — Introduction, Threats, Coral Reefs, Rising Sea Levels, and Actions — each given a unique id attribute. An internal navigation menu at the top of the article uses anchor links (href="#section-id") to allow users to jump directly to any section. A fixed-position "Go to Top" button (position: fixed) using a CSS-only <a href="#top"> approach ensures users can always return to the top without JavaScript.

Images are styled with border-radius, box-shadow, and object-fit: cover for a consistent, polished appearance. Fact highlight boxes use a left-border accent style to draw attention to key statistics. scroll-margin-top is applied to each section to prevent the sticky header from overlapping when jumping to an anchor.

Accessibility

Every image carries a descriptive alt attribute. The internal navigation <nav> uses aria-label="Page sections" so screen readers can distinguish it from the main site navigation. Heading elements follow a strict hierarchy (<h1> → <h2> → <h3>). The "Go to Top" button includes an aria-label for clarity. Text colour contrast ratios exceed WCAG AA minimums throughout.

Challenges and Lessons Learned

User Profile — JavaScript Prompt Complexity

One of the main challenges in the User Profile page was managing the flow of prompt() calls across multiple steps while allowing users to skip individual questions without breaking the profile structure. Initially, skipped prompts were leaving null values that appeared as the string "null" in the output. This was resolved by adding a conditional check: if the prompt returns null or an empty string, the field is simply not added to the profile object and not rendered to the page.

SVG Sitemap — Responsive Scaling

Positioning SVG elements precisely while keeping the diagram responsive proved tricky. Setting fixed pixel coordinates within the SVG but using a flexible viewBox allowed the diagram to scale proportionally. I also had to adjust the stroke widths and font sizes to ensure readability at smaller screen sizes.

Content Page — Anchor Offset with Sticky Header

When users clicked internal navigation links, the target section was being partially hidden behind the sticky header. This was fixed by adding scroll-margin-top: 80px to each .content-section, which pushed the scroll position down by the height of the header.

💡 Key Lesson: Testing across multiple screen sizes early — rather than only at the end — would have saved significant time during the SVG and content page development.

All pages I implemented comply with JANET (Joint Academic Network) regulations governing the publication of web pages. The key areas of compliance are as follows:

Appropriate Content: All content on my pages — including the Ocean Health content page, User Profile, and Sitemap — is educational, factual, and directly relevant to the chosen SDG (Climate Action, SDG 13). No offensive, defamatory, or misleading content is present.

Copyright and Intellectual Property: All images used are either royalty-free (sourced from Unsplash and iStockPhoto) and properly referenced, or original. No copyrighted material has been reproduced without permission.

Accessibility: Pages follow WCAG 2.1 Level AA guidelines — including meaningful alt text, sufficient colour contrast, semantic HTML, and keyboard navigation support — to ensure the site is inclusive and usable by people with disabilities.

Privacy: The User Profile page collects data only via in-browser JavaScript prompts and does not transmit or store any information server-side, ensuring user privacy.

Validation: All pages have been validated using the W3C HTML Validator with no errors reported. Evidence is provided in the Validation Page.
