Page 1

Splash Screen

Technical Description

The Splash Screen (splash.html) is a standalone, full-viewport page that does not use the shared navigation template — consistent with the coursework specification. Its layout is achieved entirely with Flexbox on the <body> element (display: flex; flex-direction: column; justify-content: center; align-items: center), centering all content vertically and horizontally without JavaScript.

A full-screen background image is applied via a background-image CSS property with a dark linear-gradient overlay rgba(0,0,0,0.72) layered on top, ensuring white text remains readable at all viewport sizes. A decorative ::before pseudo-element creates a radial green glow that pulses using a @keyframes pulse-glow animation — a purely CSS effect requiring no images or scripts.

The page includes several semantic elements: a prominent <h1> for the website title, a mission <p>, a team member list built with <div> elements styled as pill badges, a CSS spinning loader ring, a countdown text paragraph, a CSS progress bar, and a Skip Intro <button>. All content blocks carry the .fade-in class which triggers a fadeUp CSS keyframe animation with staggered animation-delay values, creating a smooth sequential reveal on load.

Key CSS techniques: Flexbox layout · background-image overlay · @keyframes (pulse-glow, spin, shrink-bar, fadeUp) · animation-delay staggering · clamp() for fluid font-size · position: relative; z-index layering

The auto-redirect is implemented with an HTML meta refresh tag (<meta http-equiv="refresh" content="4;url=home.html">) as required by the spec. JavaScript additionally manages a setInterval countdown timer that updates the on-screen text each second (4 → 3 → 2 → 1 → "Redirecting now…") and a setTimeout that fires the redirect after exactly 4 000 ms. The skipIntro() function cancels both timers via clearTimeout / clearInterval and immediately navigates to home.html.

Accessibility

The loading spinner uses role="status" and aria-label="Loading, please wait" so screen readers announce it without reading visual CSS content. The countdown paragraph uses aria-live="polite", causing assistive technologies to announce each countdown value as it changes dynamically. The progress bar wrapper carries role="progressbar" and aria-label="Redirect countdown". The Skip Intro button has a visible :focus outline style to support keyboard navigation. All team member names and the SDG badge are plain text — no icon fonts or images are used — ensuring full screen-reader access.

Page 2

Action Impact Simulator (AIS)

Technical Description

The AIS page (ais.html) uses the shared team template (header, nav, footer) and links both the global style.css and a dedicated ais.css for page-specific styles. The outer wrapper #ais-wrapper is a full-viewport container styled with min-height: 100vh, background-size: cover, and background-attachment: fixed to create a parallax effect. A semi-transparent dark overlay is rendered via the #ais-wrapper::before pseudo-element using position: fixed; inset: 0 and pointer-events: none, so it never blocks click interactions on the cards below.

The six action cards are arranged in a CSS Grid layout using grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)), which automatically creates responsive columns without media queries. Each card uses a transparent border (always present, to prevent layout shift on selection) that turns green when the .selected class is applied. The feedback panel is a centred white card styled with border-radius, box-shadow, and rgba background for a frosted appearance over the dynamic background.

Impact thresholds (defined in code comments): Low = 1–7 pts  |  Medium = 8–14 pts  |  High = 15+ pts

All JavaScript is in ais.js. The toggleCard(index) function reads a data-points attribute from each card's HTML element to retrieve its point value — avoiding hard-coded arrays and keeping HTML and JS in sync. updateFeedback() recalculates the score on every toggle, updates the DOM (score number, level label, message text), and sets wrapper.style.backgroundImage to one of three Unsplash URLs depending on the score tier — fulfilling the spec requirement for JavaScript-controlled background changes. resetAll() iterates the selected array, removes the .selected class from all cards, and calls updateFeedback() to restore the neutral state. CSS custom properties (:root variables) centralise the colour palette so all elements remain visually consistent.

Accessibility

Each action card is given role="checkbox", tabindex="0", and a dynamic aria-checked attribute (true / false) that is toggled by JavaScript alongside the visual .selected class. This means keyboard and screen-reader users receive full parity with mouse users. The handleKey(event, index) function intercepts Enter and Space key presses and calls toggleCard(), replicating native checkbox keyboard behaviour. The feedback panel carries role="region" and aria-live="polite" so its score updates are announced without disrupting the reading flow. All card images have descriptive alt text explaining the depicted scene. The tick indicator is marked aria-hidden="true" to prevent redundant announcements.

Page 3

Content Page – Reforestation

Technical Description

The content page (content_ST2.html) links both the shared style.css and a dedicated content_ST2.css. It opens with a full-width hero section using a background-image overlay technique (consistent with the AIS page), then organises the remaining body inside a .content-wrapper container.

Content is divided into semantically distinct <section> elements, each with a visually distinctive .section-tag label (OVERVIEW, THE CRISIS, THE SOLUTION, etc.) to give the reader clear structural waypoints. The "Why Forests Are Disappearing" grid and the "How You Can Help" grid both use CSS Grid with repeat(auto-fill, minmax(220px, 1fr)) for responsive tiling. The benefits section uses data-popup attributes on cards; a small inline script injects <div class="benefit-popup"> elements dynamically into each card, which are then revealed on :hover via CSS transitions — demonstrating DOM manipulation.

A sticky internal navigation bar (or go-to-top button styled with position: fixed; bottom; right) keeps orientation clear on a long-scroll page. Four or more Unsplash images are included throughout, each with descriptive alt text, consistent border-radius, and object-fit: cover to maintain aspect ratios. A photo credits section at the bottom properly attributes all external image sources with direct Unsplash links, meeting both academic referencing and the coursework's royalty-free image requirements.

Key HTML elements used: <section> · <h1>–<h3> hierarchy · <img alt="…"> · <table> (global milestones) · data-* attributes · inline <script> for DOM injection

Accessibility

All images include descriptive alt text that conveys the image content to screen readers rather than simply saying "image". Headings follow a strict <h1> → <h2> → <h3> hierarchy, giving assistive technologies an accurate document outline. The milestone table uses a <thead> with labelled <th> elements. Colour contrast between text and backgrounds follows WCAG AA guidelines (dark green on white / light green). Internal anchor links let keyboard users jump directly to each section without tabbing through all preceding content.

Challenges and Lessons Learned

Splash Screen – Auto-redirect race condition

An early challenge was ensuring the JavaScript countdown and the HTML meta refresh did not conflict. If the user clicked "Skip Intro" just before the 4-second mark, both timers could theoretically fire simultaneously. The fix was to call clearTimeout(redirectTimer) inside skipIntro() before issuing the manual redirect, guaranteeing only one navigation event occurs. This taught me the importance of managing JavaScript timer IDs explicitly rather than relying on execution order.

AIS – Background image transition

Setting background-image via JavaScript does not support CSS transitions natively (unlike background-color). I initially expected a smooth fade between levels but observed a hard cut. The workaround was to apply a CSS transition on the wrapper's opacity briefly, then change the image. This limitation also reinforced why CSS custom properties are powerful — swapping a variable triggers the cascade automatically.

Responsive grid layout

Making the six action cards reflow gracefully from three columns (desktop) to one column (mobile) without explicit breakpoints was solved using grid-template-columns: repeat(auto-fill, minmax(280px, 1fr)). This single rule replaced four separate media queries and taught me the practical power of intrinsic web design techniques.

General lesson

Code comments written at the time of implementation proved invaluable when revisiting files days later. Structured CSS using a table-of-contents comment block (as in ais.css) made editing much faster and aligns with professional codebases.

Compliance

All pages I developed comply with JANET regulations governing web page publication. Specifically:

Copyright and licensing: All images are sourced from Unsplash and used under the Unsplash License, which permits free use for any purpose without attribution (though attribution is provided voluntarily in the Photo Credits section for good academic practice). No copyrighted text has been reproduced from external sources; all written content is original.

Appropriate content: All content relates to SDG 13 (Climate Action) and presents factual, balanced, and constructive information. No offensive, discriminatory, or illegal material is present.

Accessibility: Semantic HTML, ARIA attributes, keyboard navigation, and alt text are implemented throughout, supporting WCAG 2.1 AA compliance where possible.

No personal data collection: The AIS simulator stores no user data — all selections are held in a JavaScript in-memory array and reset on page reload, so no personal information is transmitted or persisted.

Academic integrity: All HTML, CSS, and JavaScript was written by me. No pre-made templates were downloaded from the internet, consistent with the coursework requirement. Where external resources (Unsplash, Google Fonts) are used, they are properly referenced.
